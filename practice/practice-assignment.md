# Навчальна практика

**Дисципліна:** Сучасні технології Internet-програмування<br>
**Спеціальність:** Інженерія програмного забезпечення, 3 курс<br>
**Тривалість:** 5 днів<br>
**Форма роботи:** індивідуальна

---

## Мета практики

Закріпити знання та практичні навички, набуті протягом курсу, через розробку індивідуального full-stack веб-застосунку з використанням React та Nest.js.

---

## Загальний опис

Кожен студент самостійно розробляє веб-застосунок на обрану тему. Проєкт складається з двох частин: серверної (REST API на Nest.js) та клієнтської (SPA на React). Робота ведеться в Git-репозиторії зі щоденними комітами та публікацією на GitHub.

**Технологічний стек:**

- **Backend:** Node.js, Nest.js, TypeScript
- **Frontend:** React (Vite), TypeScript
- **Система контролю версій:** Git + GitHub
- **Інструменти:** VS Code, cURL / Postman / Bruno

**Структура репозиторію:** frontend та backend розміщуються в одному репозиторії (monorepo) в окремих папках, наприклад:

```
my-project/
├── backend/      # Nest.js проєкт
├── frontend/     # React проєкт
└── README.md
```

## Розклад роботи за днями

### День 1: Планування та Backend основа

**Задачі:**

1. Обрати тему проєкту
2. Створити публічний репозиторій на GitHub (якщо приватний — додайте викладача як collaborator)
3. Ініціалізувати Nest.js проєкт (`@nestjs/cli`)
4. Спроєктувати сутності та ендпоінти (описати в README або окремому файлі)
5. Створити базову структуру: модулі, контролери, сервіси
6. Реалізувати in-memory сховище (масиви/Map) для 1-2 основних ресурсів
7. Реалізувати базові GET та POST ендпоінти
8. Перевірити роботу API через cURL, Postman або Bruno

**Очікуваний результат (push на GitHub):**

- Ініціалізований Nest.js проєкт з правильною структурою
- Мінімум 1-2 ресурси з GET/POST ендпоінтами
- API відповідає коректними даними

---

### День 2: Backend — повний CRUD та валідація

**Задачі:**

1. Реалізувати повний CRUD (Create, Read, Update, Delete) для всіх сутностей
2. Створити DTO (Data Transfer Objects) для кожної операції
3. Додати валідацію вхідних даних через `class-validator` та `class-transformer`
4. Налаштувати глобальний `ValidationPipe`
5. Протестувати всі ендпоінти через cURL/Postman/Bruno

**Очікуваний результат (push на GitHub):**

- Повноцінний REST API з усіма CRUD-операціями
- Валідація вхідних даних з інформативними повідомленнями про помилки
- Коректні HTTP-статуси відповідей (200, 201, 400, 404 тощо)

---

### День 3: Frontend — React застосунок

**Задачі:**

1. Ініціалізувати React проєкт з Vite та TypeScript
2. Створити структуру компонентів та сторінок
3. Налаштувати маршрутизацію (React Router): мінімум 3 сторінки
4. Підключитися до API (fetch або axios)
5. Реалізувати відображення списку сутностей
6. Реалізувати форми створення/редагування з валідацією (React Hook Form + Zod або ручна валідація)
7. Додати навігацію між сторінками

**Очікуваний результат (push на GitHub):**

- Працюючий React-застосунок з маршрутизацією
- Дані завантажуються з API та відображаються в інтерфейсі
- Форми працюють з валідацією

---

### День 4: Інтеграція, стилізація та фіналізація

**Задачі:**

1. Реалізувати повну інтеграцію: всі CRUD-операції працюють через інтерфейс
2. Стилізувати застосунок (CSS/SCSS/CSS Modules або UI-бібліотека)
3. Додати обробку помилок та стани завантаження в UI
4. Підготувати README.md:
   - Опис проєкту та його функціональності
   - Використані технології
   - Інструкція з встановлення та запуску (backend та frontend)
   - Скріншоти інтерфейсу
5. Фінальна перевірка та виправлення багів

**Очікуваний результат (push на GitHub):**

- Повністю працюючий full-stack застосунок
- Стилізований інтерфейс
- README.md з повним описом проєкту

---

### День 5: Оформлення звіту та захист

**Задачі:**

1. Оформити звіт з практики (структура нижче)
2. Підготувати демонстрацію проєкту (запуск та показ основного функціоналу)
3. Захист проєкту перед викладачем

## Мінімальні технічні вимоги

### Backend (Nest.js)

- Мінімум **2 сутності** (наприклад: posts + comments, books + authors)
- **Повний CRUD** для кожної сутності (GET all, GET by id, POST, PUT/PATCH, DELETE)
- **DTO** для операцій створення та оновлення
- **Валідація** вхідних даних (`class-validator`)
- Правильна структура проєкту (modules, controllers, services)
- In-memory сховище даних (масиви або Map)

### Frontend (React)

- React + TypeScript (створено через Vite)
- Мінімум **3 сторінки** з маршрутизацією (React Router)
- Підключення до API бекенду (fetch або axios)
- **Відображення даних** з сервера (списки, детальна інформація)
- **Форми** створення та редагування з валідацією
- **Видалення** записів через інтерфейс
- Стани завантаження та обробка помилок в UI
- Базова стилізація

**Рекомендація:** для спрощення роботи зі стилями та візуалізацією варто використати бібліотеку UI-компонентів:

- [Material UI (MUI)](https://mui.com/) — найпопулярніша React UI бібліотека
- [Ant Design](https://ant.design/) — комплексна бібліотека з великою кількістю компонентів
- [Chakra UI](https://chakra-ui.com/) — проста та гнучка бібліотека з фокусом на доступність
- [Mantine](https://mantine.dev/) — сучасна бібліотека з великим набором хуків та компонентів

### Git та GitHub

- Публічний репозиторій на GitHub (якщо приватний — додайте викладача як collaborator)
- Осмислені коміти з описовими повідомленнями (не "fix", "update")
- Мінімум **1 коміт на кожен робочий день**
- `README.md` з описом проєкту, технологіями та інструкцією запуску

## Необов'язкові можливості

Наступні пункти не є обов'язковими, але їх реалізація продемонструє глибше розуміння матеріалу:

- **PostgreSQL + TypeORM** замість in-memory сховища (entities, repositories, relations)
- **TanStack Query** для управління серверним станом на frontend
- **Управління станом** (React Context або Zustand)
- **Пагінація та фільтрація** на бекенді та фронтенді
- **Завантаження файлів** (зображення, аватари тощо)
- Оптимізація React (React.memo, useMemo, useCallback, code splitting)
- Адаптивний дизайн (responsive)

## Вимоги до звіту з практики

Звіт оформлюється як окремий документ та має містити:

1. **Титульна сторінка**
2. **Вступ** (мета практики, обрана тема, коротко про технології)
3. **Постановка задачі** (функціональні вимоги до застосунку, опис сутностей та їх зв'язків)
4. **Реалізація**
   - Структура проєкту (backend та frontend)
   - Опис основних модулів та компонентів
   - Опис API ендпоінтів (таблиця: метод, шлях, опис)
   - Фрагменти ключового коду з поясненнями
5. **Скріншоти роботи застосунку** (основні сторінки та функції)
6. **Висновки** (що було зроблено, які навички набуто, труднощі та їх вирішення)
7. **Посилання на GitHub-репозиторій**

## Корисні команди для початку роботи

### Backend (Nest.js)

```bash
# Встановлення CLI
npm i -g @nestjs/cli

# Створення проєкту
nest new project-name

# Генерація ресурсу (module + controller + service)
nest generate resource resource-name

# Встановлення валідації
npm i class-validator class-transformer

# Запуск у режимі розробки
npm run start:dev
```

### Frontend (React + Vite)

```bash
# Створення проєкту
npm create vite@latest project-name -- --template react-ts

# Встановлення залежностей
cd project-name && npm install

# Встановлення роутера
npm i react-router-dom

# Запуск у режимі розробки
npm run dev
```

### Git

```bash
# Ініціалізація та перший коміт
git init
git add .
git commit -m "Initial project setup"

# Підключення до GitHub
git remote add origin https://github.com/username/repo-name.git
git push -u origin main
```

> **Важливо:** переконайтесь, що `.gitignore` містить `node_modules/`, `.env` та інші файли, які не мають потрапляти в репозиторій. Nest.js та Vite генерують `.gitignore` автоматично, але перевірте його наявність у кореневій папці проєкту.
